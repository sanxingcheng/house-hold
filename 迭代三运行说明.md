# 迭代三运行说明

## 环境要求

- **JDK 21**
- **Node 18+**（前端）
- **MySQL 8**（库 `household`）
- **Maven 3.9+**
- （可选）**Redis**：启用缓存 L2 层（不启动 Redis 仅使用 Caffeine 本地缓存，不影响功能）
- （可选）**Nacos**：服务发现与配置中心；不使用可参考下方「无 Nacos 运行」方式

## 1. 数据库

若已完成迭代一/二，数据库 `household` 已存在，无需重建。首次使用时：

```sql
CREATE DATABASE IF NOT EXISTS household DEFAULT CHARACTER SET utf8mb4;
```

迭代三新增的 `account` 和 `wealth_snapshot` 两张表会在 `house-hold-wealth` 服务启动时由 JPA 自动创建（`ddl-auto: update`），无需手动建表。

如需手动执行建表 SQL，参见 `house-hold-wealth/src/main/resources/schema.sql`。

修改 `house-hold-wealth/src/main/resources/application.yml` 中的 `spring.datasource.url`、`username`、`password` 为你的 MySQL 配置（默认 `root / Admin@123`）。

## 2. 无 Nacos 运行（开发）

在项目根目录 `house-hold` 下按顺序执行：

### 2.1 启动 auth-user 服务（端口 8081）

```bash
mvn -pl house-hold-auth-user spring-boot:run
```

### 2.2 启动 wealth 服务（端口 8082）

```bash
mvn -pl house-hold-wealth spring-boot:run
```

### 2.3 启动 Gateway（端口 8080）

使用 dev 配置直连后端服务（免 Nacos）：

```bash
mvn -pl house-hold-gateway spring-boot:run -Dspring-boot.run.profiles=dev
```

### 2.4 启动前端

```bash
cd house-hold-web
npm install
npm run dev
```

访问 http://localhost:5173

## 3. 使用 Nacos 运行

1. 启动 Nacos（默认 8848）
2. 启动 auth-user：`mvn -pl house-hold-auth-user spring-boot:run`
3. 启动 wealth：`mvn -pl house-hold-wealth spring-boot:run`
4. 启动 Gateway：`mvn -pl house-hold-gateway spring-boot:run`（无需 dev 配置）
5. 启动前端：`cd house-hold-web && npm install && npm run dev`

## 4. 服务端口一览

| 服务 | 端口 | 说明 |
|------|------|------|
| house-hold-gateway | 8080 | API 网关，统一入口 |
| house-hold-auth-user | 8081 | 用户认证与家庭管理 |
| house-hold-wealth | 8082 | 财富管理（迭代三新增） |
| house-hold-web | 5173 | 前端开发服务器 |

## 5. 迭代三新增接口

所有接口通过 Gateway（8080）访问，需携带 `Authorization: Bearer <token>` 请求头。

### 5.1 账户管理

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| GET | `/wealth/accounts` | 查询个人账户列表 | — |
| POST | `/wealth/accounts` | 新增账户 | `{ "accountName", "accountType", "balance", "currency?" }` |
| PUT | `/wealth/accounts/{id}` | 编辑账户 | `{ "accountName?", "accountType?", "balance?", "currency?" }` |
| DELETE | `/wealth/accounts/{id}` | 删除账户 | — |

**accountType 可选值**：`SAVING`（储蓄卡）、`CREDIT_CARD`（信用卡）、`STOCK`（股票）、`FUND`（基金）、`ALIPAY`（支付宝）、`WECHAT`（微信）、`OTHER`（其他）

**balance 单位**：分（如 50000 表示 ¥500.00）

### 5.2 资产汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/wealth/summary/user` | 个人资产汇总 |
| GET | `/wealth/summary/family` | 家庭资产汇总（需已加入家庭） |

返回示例：
```json
{
  "ownerId": "1000000000001",
  "ownerType": "USER",
  "totalAssets": 8000000,
  "totalLiabilities": 1500000,
  "netWorth": 6500000,
  "snapshotTime": "2026-03-02T12:30:00"
}
```

### 5.3 历史趋势

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| GET | `/wealth/history/user` | `from`, `to`（YYYY-MM-DD） | 个人资产历史 |
| GET | `/wealth/history/family` | `from`, `to`（YYYY-MM-DD） | 家庭资产历史 |

返回示例：
```json
[
  { "snapshotDate": "2026-02-01", "netWorth": 5000000 },
  { "snapshotDate": "2026-03-01", "netWorth": 6500000 }
]
```

## 6. 前端页面

| 路径 | 页面 | 说明 |
|------|------|------|
| `/dashboard` | 概览 | 新增个人资产 / 家庭资产卡片 |
| `/wealth/accounts` | 账户管理 | 账户 CRUD，卡片式列表，信用卡负债红色展示 |
| `/wealth/history` | 资产趋势 | 个人/家庭 Tab 切换，30/90/180 天折线图 |

导航栏新增「账户管理」和「资产趋势」入口。

## 7. 缓存说明

默认启用 Caffeine 本地缓存（L1），若 Redis 可用则自动启用 L2。

- **账户列表**：L1 60s / L2 300s + 随机偏移
- **资产汇总**：L1 30s（通过 Caffeine 全局 TTL 60s）/ L2 60s + 随机偏移
- 账户变更时自动清除相关缓存（Cache-Aside 失效模式）

如需关闭缓存，修改 `house-hold-wealth/src/main/resources/application.yml`：

```yaml
cache:
  enabled: false
```

## 8. 快速验证流程

1. 注册并登录用户
2. 进入「账户管理」，新增一个储蓄卡账户（余额 500 元 → 输入 500）
3. 新增一个信用卡账户（账单 100 元 → 输入 100）
4. 回到「概览」页，查看个人资产卡片显示净资产 ¥400.00
5. 进入「资产趋势」，查看折线图出现今日数据点
