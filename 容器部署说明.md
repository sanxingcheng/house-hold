# 容器与 Kubernetes 部署说明

本文说明在 Docker 与 Kubernetes 环境下部署 house-hold 各服务的方式。

## 一、前置依赖

- **Docker**：用于构建镜像与本地/单机运行
- **Kubernetes**：可选，用于编排（需 Nacos、MySQL、Redis、Kafka 等中间件可访问）
- **中间件**：MySQL（`household_auth`、`household_wealth` 两库）、Redis、Nacos、Kafka（可选）。容器场景下需先部署或使用已有实例。

## 二、Docker 镜像构建

所有镜像构建均在**项目根目录** `house-hold/` 下执行（前端除外）。

### 2.1 后端服务（Gateway / Auth-User / Wealth）

```bash
# 进入项目根目录
cd house-hold

# 网关
docker build -f docker/Dockerfile.gateway -t house-hold-gateway:latest .

# 用户与家庭服务
docker build -f docker/Dockerfile.auth-user -t house-hold-auth-user:latest .

# 财富服务
docker build -f docker/Dockerfile.wealth -t house-hold-wealth:latest .
```

### 2.2 前端（Vue + Nginx）

前端需指定 API 基地址（与后端访问方式一致），例如同域时使用相对路径 `/api` 或留空。

```bash
# 在项目根目录下指定前端子目录为上下文，并传入 API 基地址（与 Ingress/网关路径一致）
docker build -f house-hold-web/Dockerfile \
  --build-arg VITE_API_BASE_URL= \
  -t house-hold-web:latest \
  house-hold-web
```

若通过 Ingress 将网关暴露为同域（如 `https://example.com/auth`、`/family`、`/wealth`），则 `VITE_API_BASE_URL` 留空即可；若 API 为独立域名可设为 `https://api.example.com`。

## 三、仅 Docker 运行（单机）

确保 MySQL、Redis、Nacos 已启动且网络可达（同机可用 `host.docker.internal` 或宿主机 IP）。

```bash
# 示例：网关（端口 8080）
docker run -d --name house-hold-gateway \
  -e SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR=host.docker.internal:8848 \
  -e SPRING_DATA_REDIS_HOST=host.docker.internal \
  -p 8080:8080 \
  house-hold-gateway:latest

# 用户与家庭服务（端口 8081）
docker run -d --name house-hold-auth-user \
  -e SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR=host.docker.internal:8848 \
  -e SPRING_DATA_REDIS_HOST=host.docker.internal \
  -e SPRING_DATASOURCE_URL="jdbc:mysql://host.docker.internal:3306/household_auth?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai" \
  -e SPRING_DATASOURCE_USERNAME=root \
  -e SPRING_DATASOURCE_PASSWORD=Admin@123 \
  -e SPRING_KAFKA_BOOTSTRAP_SERVERS=host.docker.internal:9092 \
  -p 8081:8081 \
  house-hold-auth-user:latest

# 财富服务（端口 8082）：数据源在 jar 内 shardingsphere.yaml 中，默认 localhost；容器内需能访问 MySQL，可通过 Nacos 覆盖配置或自建镜像替换 yaml
docker run -d --name house-hold-wealth \
  -e SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR=host.docker.internal:8848 \
  -e SPRING_DATA_REDIS_HOST=host.docker.internal \
  -p 8082:8082 \
  house-hold-wealth:latest

# 前端（端口 80）
docker run -d --name house-hold-web -p 80:80 house-hold-web:latest
```

访问 `http://localhost` 为前端；接口通过网关 `http://localhost:8080`（或前端 nginx 内代理到网关）访问。

## 四、Kubernetes 部署

### 4.1 准备

1. **创建数据库**：在 MySQL 中执行 `scripts/init-databases.sql`，创建 `household_auth`、`household_wealth`。
2. **中间件**：在集群内或集群外提供 Nacos、Redis、MySQL、Kafka（可选），并记下服务地址/端口。
3. **修改 K8s 配置**：编辑 `k8s/configmap.yaml`、`k8s/secret.yaml`，将 Nacos/Redis/MySQL 等地址、密码改为实际值。

### 4.2 部署顺序

```bash
# 1. 命名空间与通用配置
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/secret.yaml

# 2. 业务服务（需先有 Nacos、Redis、MySQL）
kubectl apply -f k8s/gateway-deployment.yaml
kubectl apply -f k8s/auth-user-deployment.yaml
kubectl apply -f k8s/wealth-deployment.yaml
kubectl apply -f k8s/web-deployment.yaml

# 3. 对外访问（二选一或同时使用）
# 方式 A：Ingress（需集群已装 Ingress Controller，并修改 ingress.yaml 中的 host）
kubectl apply -f k8s/ingress.yaml

# 方式 B：Service 使用 NodePort 或 LoadBalancer，直接暴露网关与前端端口
# 可编辑对应 Service，设置 type: NodePort 或 LoadBalancer
```

### 4.3 配置说明

- **ConfigMap `house-hold-config`**：提供 Nacos、Redis、MySQL、Kafka 等地址，各 Deployment 通过环境变量引用。
- **Secret `house-hold-secret`**：存放 MySQL 密码等敏感信息，生产务必替换为安全值。
- **auth-user**：通过 `SPRING_DATASOURCE_*` 注入数据源，指向 `household_auth`。
- **wealth**：当前数据源写在 jar 内 `shardingsphere.yaml` 中。在 K8s 中可：
  - 使用 **Nacos 配置**为 `house-hold-wealth` 覆盖数据源，或
  - 构建镜像时在镜像内替换 `shardingsphere.yaml` 中的 `jdbcUrl`/用户名密码为集群内 MySQL 地址。

### 4.4 访问与排查

- **Ingress**：若使用 `k8s/ingress.yaml`，将 `house-hold.local` 改为你的域名，并在本机 hosts 或 DNS 中指向 Ingress 的 IP；浏览器访问 `http://house-hold.local` 即可打开前端，接口同域。
- **查看 Pod**：`kubectl get pods -n house-hold`
- **查看日志**：`kubectl logs -f deployment/house-hold-gateway -n house-hold`
- **健康检查**：各服务提供 `/actuator/health`，Deployment 中已配置 liveness/readiness 探针。

## 五、文件清单

| 路径 | 说明 |
|------|------|
| `docker/Dockerfile.gateway` | 网关镜像构建 |
| `docker/Dockerfile.auth-user` | 用户与家庭服务镜像构建 |
| `docker/Dockerfile.wealth` | 财富服务镜像构建 |
| `house-hold-web/Dockerfile` | 前端镜像构建（Nginx 托管） |
| `house-hold-web/nginx.conf` | 前端 Nginx 配置 |

| `k8s/ingress.yaml` | Ingress 统一入口（可选） |
| `scripts/init-databases.sql` | 初始化 household_auth / household_wealth 库 |

## 六、注意与建议

- 生产环境请为镜像打明确版本号（如 `house-hold-gateway:1.0.0`），并使用私有镜像仓库与 `imagePullSecrets`。
- JWT 密钥、数据库密码等务必通过 Secret 或外部配置管理，勿写死在镜像或 ConfigMap 明文。
- 若启用 Kafka，需保证 auth-user 能连上 `KAFKA_BOOTSTRAP_SERVERS`；未配置时 auth-user 仍可运行，仅不发送领域事件。
- wealth 服务依赖 Nacos 发现 auth-user 以调用 REST 校验管理员；部署时确保 Nacos 中已注册 auth-user、wealth、gateway。
