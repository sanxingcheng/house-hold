# 迭代四运行说明

## 环境要求

- **JDK 21**
- **Node 18+**（前端）
- **MySQL 8**（库 `household`）
- **Maven 3.9+**
- **Redis**（必需：Session 管理、分布式锁、缓存 L2 层）
- （可选）**Nacos**：服务发现与配置中心；不使用可参考下方「无 Nacos 运行」方式

## 1. 数据库

### 1.1 基础库

若已完成前几个迭代，数据库 `household` 已存在。首次使用时：

```sql
CREATE DATABASE IF NOT EXISTS household DEFAULT CHARACTER SET utf8mb4;
```

### 1.2 迭代四 DDL — account 分表

迭代四对 `account` 表进行了 ShardingSphere 水平分表（按 `user_id % 4` 分为 4 张物理表），**需手动执行建表 SQL**：

```sql
USE household;

-- 创建 4 张分表
CREATE TABLE IF NOT EXISTS account_0 (
  id            BIGINT PRIMARY KEY,
  user_id       BIGINT NOT NULL,
  family_id     BIGINT DEFAULT NULL,
  account_name  VARCHAR(64) NOT NULL,
  account_type  VARCHAR(32) NOT NULL,
  balance       BIGINT NOT NULL DEFAULT 0,
  currency      VARCHAR(8) NOT NULL DEFAULT 'CNY',
  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_user_id   (user_id),
  KEY idx_family_id (family_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS account_1 LIKE account_0;
CREATE TABLE IF NOT EXISTS account_2 LIKE account_0;
CREATE TABLE IF NOT EXISTS account_3 LIKE account_0;

-- wealth_snapshot 表保持不变（不分表）
CREATE TABLE IF NOT EXISTS wealth_snapshot (
  id                BIGINT PRIMARY KEY,
  owner_type        VARCHAR(16) NOT NULL,
  owner_id          BIGINT NOT NULL,
  total_assets      BIGINT NOT NULL DEFAULT 0,
  total_liabilities BIGINT NOT NULL DEFAULT 0,
  net_worth         BIGINT NOT NULL DEFAULT 0,
  snapshot_date     DATE NOT NULL,
  created_at        DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_owner_date (owner_type, owner_id, snapshot_date),
  KEY idx_owner (owner_type, owner_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

如果有存量数据在原 `account` 表中，需手动迁移：

```sql
INSERT INTO account_0 SELECT * FROM account WHERE user_id % 4 = 0;
INSERT INTO account_1 SELECT * FROM account WHERE user_id % 4 = 1;
INSERT INTO account_2 SELECT * FROM account WHERE user_id % 4 = 2;
INSERT INTO account_3 SELECT * FROM account WHERE user_id % 4 = 3;
-- 确认迁移完成后可删除原表
-- DROP TABLE account;
```

### 1.3 ShardingSphere 配置

ShardingSphere 的分表配置位于 `house-hold-wealth/src/main/resources/shardingsphere.yaml`，默认连接 `127.0.0.1:3306/household`。如需修改数据库地址，请同时修改该文件和 `application.yml`。

## 2. 无 Nacos 运行（开发）

在项目根目录 `house-hold` 下按顺序执行：

### 2.1 启动 auth-user 服务（端口 8081）

```bash
mvn -pl house-hold-auth-user spring-boot:run
```

### 2.2 启动 wealth 服务（端口 8082）

```bash
mvn -pl house-hold-wealth spring-boot:run
```

### 2.3 启动 Gateway（端口 8080）

```bash
mvn -pl house-hold-gateway spring-boot:run -Dspring-boot.run.profiles=dev
```

### 2.4 启动前端

```bash
cd house-hold-web
npm install
npm run dev
```

访问 http://localhost:5173

## 3. 使用 Nacos 运行

1. 启动 Nacos（默认 8848）
2. 启动 Redis（默认 6379）
3. 启动 auth-user：`mvn -pl house-hold-auth-user spring-boot:run`
4. 启动 wealth：`mvn -pl house-hold-wealth spring-boot:run`
5. 启动 Gateway：`mvn -pl house-hold-gateway spring-boot:run`
6. 启动前端：`cd house-hold-web && npm install && npm run dev`

## 5. 迭代四新增功能

### 5.1 监控端点

每个服务均暴露 Actuator 监控端点（通过各服务端口访问）：

| 端点 | 路径 | 说明 |
|------|------|------|
| 健康检查 | `/actuator/health` | 服务健康状态（UP/DOWN），含 Redis 等组件状态 |
| Prometheus 指标 | `/actuator/prometheus` | Prometheus 格式的 metrics 数据 |
| Metrics | `/actuator/metrics` | 可用指标列表 |
| Info | `/actuator/info` | 应用基本信息 |

示例：

```bash
curl http://localhost:8081/actuator/health
curl http://localhost:8082/actuator/prometheus
```

### 5.2 定时快照任务

- **执行时间**：每日凌晨 2:00（cron: `0 0 2 * * ?`）
- **行为**：遍历所有有账户的用户，自动生成当日快照
- **分布式锁**：多实例部署时仅一个实例执行（Redisson 锁 `household:lock:snapshot-scheduler`）
- **配置**：`house-hold-wealth/src/main/resources/application.yml`

```yaml
snapshot:
  scheduler:
    enabled: true          # 设为 false 可禁用
    cron: "0 0 2 * * ?"   # 可自定义执行时间
```

### 5.3 结构化日志

- **开发环境**（默认）：彩色控制台输出，`com.household` 包为 DEBUG 级别
- **生产环境**：`--spring.profiles.active=prod` 启动，JSON 格式日志输出
- **请求追踪**：每个请求自动生成 `traceId` 放入 MDC，日志中可见
- **Gateway 访问日志**：记录每个请求的路由目标、状态码、耗时

### 5.4 登出接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/logout` | 登出（清除 Redis Session），需携带 `Authorization` 头 |

前端退出时会自动调用此接口。

## 6. 前端 UI 变化

迭代四全面引入 **Element Plus** 组件库和 **ECharts** 图表库，所有页面已重构：

| 页面 | 主要变化 |
|------|---------|
| 登录/注册 | Element Plus 表单 + 渐变背景卡片布局 |
| 概览 | 资产卡片使用 `ElStatistic` 展示数字 |
| 个人信息 | Element Plus 表单 + 日期选择器 |
| 家庭 | Element Plus 表格展示成员，标签展示角色 |
| 账户管理 | Element Plus 表格 + 对话框表单 + 气泡确认删除 |
| 资产趋势 | **ECharts 折线图**替换原 Canvas 自绘图表 |
| 导航栏 | Element Plus 菜单 + 用户头像下拉 |

## 7. 快速验证流程

1. 执行迭代四 DDL（创建 account_0~3 分表）
2. 启动 Redis → 启动后端 3 个服务 → 启动前端
3. 注册并登录
4. 进入「账户管理」→ 新增账户 → 确认 UI 为 Element Plus 风格
5. 查看「资产趋势」→ 确认为 ECharts 折线图
6. 访问 `http://localhost:8081/actuator/health` → 确认返回 `UP`
7. 访问 `http://localhost:8082/actuator/prometheus` → 确认返回 Prometheus 指标
8. 查看后端控制台日志 → 确认包含 `traceId` 和请求耗时
9. 退出登录 → 确认前端调用 `/auth/logout` 接口
